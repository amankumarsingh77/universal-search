# Frontend

## Wails Bindings

All Go `App` methods are callable from TypeScript as:
```typescript
import * as App from "../wailsjs/go/main/App";
await App.SearchWithFilters(raw, semanticQuery, denyList);
await App.GetFolders();
await App.ParseQuery(raw);
```

Generated bindings live in `frontend/wailsjs/go/`. Do not edit these files — they are regenerated by `wails dev`.

## Search State Machine (`searchReducer.ts`)

A finite state machine managing the search bar's chip state:
- `pending` — chip typed but not committed
- `active` — chip committed, included in filter

State transitions:
- User types `kind:image` → token enters `pending` state
- User presses Enter or space → token moves to `active`
- User clicks chip × → token removed from `active`
- `ParseQuery` result arrives → chips reconciled against returned `FilterChipDTO` array

## Filter Chips (`FilterChip.tsx`)

Renders one chip per active `FilterChipDTO`. Each chip displays `label: value`. Clicking × dispatches a remove action to `searchReducer`.

## Preview Panel

Context-aware, right-side panel. File-type dispatch:
- Text / code → rendered with syntax highlighting, up to 8KB via `GetFilePreview`
- Image → `<img>` tag with file:// URL
- PDF → browser PDF viewer
- Video → `<video>` tag
- Binary / unsupported → "Preview not available"

## Indexing Progress Bar

Collapsible bar at the bottom. Driven by polling `GetIndexStatus()` every 500ms when visible. Shows `Indexed N / Total`, rate-limited countdown when `QuotaPaused = true`.

## Offline Indicator

Shown in the search bar when `GetNLQueryEnabled()` returns false. Tooltip: "Natural language search disabled — check API key".
